# 서비스팩토리 오픈소스 출처 노트

## 사용 원칙
- 이 문서는 서비스화 아이디어와 설치 대행 후보를 정리하기 위한 출처 노트입니다.
- 오픈소스 코드를 이 폴더로 복사하지 않습니다.
- 실제 납품 전에는 각 프로젝트의 최신 라이선스 확인, 상표 사용 조건, self-host 지원 범위, 보안 업데이트 상태를 다시 봅니다.
- 고객에게는 소프트웨어 자체 판매가 아니라 설치, 설정, 운영, 교육, 유지보수 서비스를 판매합니다.

## 라이선스 확인 체크
- [ ] LICENSE 파일과 상업적 사용 가능 범위 확인
- [ ] AGPL, SSPL, custom license 등 네트워크 사용 조건 확인
- [ ] 고객 서버에 설치하는 경우 소스 공개 의무 여부 확인
- [ ] SaaS로 재판매하는 경우 프로젝트 정책 확인
- [ ] 로고, 프로젝트명, 상표 사용 가능 범위 확인
- [ ] 보안 업데이트와 백업 책임을 계약서에 명시

## 바로 서비스화하기 좋은 순서
1. 직접 제작형 문의/견적/UTM/CSV 도구: 계정 없이 빠르게 납품 가능.
2. Activepieces 또는 Umami: 작은 자동화/분석부터 판매 가능.
3. Chatwoot 또는 FreeScout: CS 문제가 분명한 고객에게 제안하기 좋음.
4. Twenty 또는 EspoCRM: B2B 영업 고객에게 고가 세팅으로 판매.
5. Metabase/Appsmith: 데이터가 이미 있는 고객에게 고가 구축으로 판매.
