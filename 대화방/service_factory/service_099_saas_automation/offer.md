# SaaS 문의 자동화 서비스

서비스 번호: 099
원본 ID: svc-349
rank / tier / score: 99 / B / 87
카테고리 / 니치: automation / SaaS
권장 채널: web-service
플랫폼: Gumroad
결제 시스템: Stripe Payment Links
가격 앵커: ₩300,000 setup + ₩90,000/mo

## 오퍼
Activepieces 또는 로컬 도구를 활용해 SaaS의 문의 자동화 흐름을 세팅하고 운영 문서까지 납품한다.

## 고객 문제
예약 변경과 노쇼 대응이 수작업이다

## 첫 실행 기준
SaaS의 현재 업무 화면 1개와 샘플 데이터 10개를 받아 첫 MVP 범위를 고정한다.

## 납품물
- 자동화 흐름도
- 실패 알림 설계
- 재시도 규칙
- 고객 계정/API 권한 체크리스트
- 샘플 실행 로그
- 유지보수 점검표

## 작업 단계
- 1. 현재 문의/업무 채널과 담당자 흐름을 정리한다.
- 2. 자동화 트리거, 조건, 예외 처리, 알림 채널을 설계한다.
- 3. 샘플 데이터로 성공/실패/중복 상황을 테스트한다.
- 4. 운영자가 직접 중단, 재실행, 변경 요청을 할 수 있게 문서화한다.

## 필수 보강
실패 알림, 재시도 규칙, 고객 계정 권한 체크리스트를 추가한다.

## 약점과 방어
고객 계정/API 권한과 실패 알림 설계가 부족하면 납품 후 유지보수 부담이 커진다.

## 결제/채널 주의
- Stripe Payment Links: 서비스 착수금, 월 유지보수, 단일 결제 링크. 주의: 디지털 앱 기능을 모바일 앱 안에서 팔 때는 앱스토어 정책을 따로 본다.
- Stripe Checkout: 자체 웹서비스의 장바구니/구독. 주의: 개발 연동과 웹훅 검증이 필요하다.
- Paddle Checkout: 글로벌 디지털 상품/소프트웨어. 주의: 판매자 승인과 상품 심사가 필요하다.
- Lemon Squeezy Checkout: 라이선스/다운로드/구독. 주의: 계정 승인과 지원 국가 확인 필요.
- web: 정적 카탈로그 → 결제 링크 → 납품 ZIP/온보딩 폼 → 월 유지보수. 주의: 실제 결제 링크는 사용자 소유 계정에서 생성한다.

## 환불 및 성과 안내
실제 결제는 사용자 소유 계정에서만 진행합니다. 납품 범위, 환불 조건, 고객 자료 제공 지연 시 일정 변경 조건을 사전에 안내합니다. 이 서비스는 세팅과 운영 문서를 납품하지만 매출, 문의, 수익을 보장하지 않습니다.
