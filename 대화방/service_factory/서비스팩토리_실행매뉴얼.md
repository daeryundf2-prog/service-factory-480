# 서비스팩토리 실행매뉴얼

## 목적
이 매뉴얼은 오픈소스와 로컬 제작 도구를 이용해 설치 대행, 초기 세팅, 월 유지보수 형태의 서비스를 빠르게 만드는 실행 문서입니다.

## 핵심 원칙
- 오픈소스 라이선스, 상표, 재판매 가능 범위를 먼저 확인한다.
- "수익 보장"이 아니라 "업무 세팅, 설치, 운영 지원"을 판매한다.
- 첫 납품은 1개 문제, 1개 고객군, 1개 결과 화면으로 제한한다.
- 고객의 결제 계정, 서버 계정, 도메인, 개인정보 처리 책임은 고객 소유로 둔다.

## 1일차: 고를 서비스
1. `services.json`에서 고객군이 명확한 후보 3개를 고른다.
2. 난이도 `low` 또는 `medium`부터 시작한다.
3. 고객에게 바로 보여줄 결과가 있는지 확인한다.
4. 가격은 초기 세팅비와 월 유지보수비를 분리한다.

추천 빠른 후보:
- Activepieces: 시트, 이메일, 알림 자동화.
- Twenty: 리드와 영업 파이프라인 CRM.
- Chatwoot: 웹채팅과 고객상담 인박스.
- Formbricks: 고객 설문과 이탈 피드백.
- Metabase: 매출, 광고, 리드 대시보드.

## 2일차: 판매용 샘플 만들기
1. 고객 업종 1개를 정한다.
2. 데모 데이터 10개를 만든다.
3. 화면 캡처 3장을 만든다.
4. "전/후" 설명을 쓴다.
5. 고객에게 줄 첫 문장을 만든다.

예시:
- 전: 문의가 이메일, 카톡, 전화로 흩어져 누락된다.
- 후: 모든 문의가 Chatwoot 인박스에 모이고, 라벨과 담당자가 자동 지정된다.

## 3일차: 첫 판매
1. 기존 고객, 지역 매장, 프리랜서 커뮤니티에서 30곳을 찾는다.
2. `서비스팩토리_영업패키지.md`의 문구를 업종명으로 바꾼다.
3. 10곳씩 다른 제목으로 보낸다.
4. 답변이 온 경우 바로 무료 진단 15분을 제안한다.
5. 진단 후에는 "이번 주 안에 끝나는 1개 세팅"만 견적낸다.

## 4일차 이후: 월 유지보수로 바꾸기
- 알림 실패 확인
- 보안 업데이트
- 백업 확인
- 월간 리포트 캡처
- 고객 요청 1시간 포함

## 오픈소스별 서비스화 주의
### Activepieces
반복 업무 자동화에 좋다. 앱 권한과 API 제한을 고객에게 설명한다.

### Twenty
CRM 커스텀에 좋다. DB와 Redis 운영이 필요하므로 구축비를 낮게 잡지 않는다.

### Chatwoot
상담 채널 통합에 좋다. 메신저 API, 개인정보, 알림 정책을 계약서에 포함한다.

### Formbricks
설문과 고객 피드백에 좋다. 쿠키, 추적, 수집 항목 고지를 분명히 한다.

### Metabase
대시보드에 좋다. 데이터 원천 품질이 낮으면 먼저 정리 서비스를 판다.

## 설치형 서비스 공통 체크리스트
- [ ] 라이선스와 상업적 사용 가능 범위 확인
- [ ] 서버 비용과 백업 책임 분리
- [ ] 관리자 계정은 고객 이메일로 생성
- [ ] 기본 비밀번호 제거
- [ ] 개인정보 처리 위탁 또는 처리방침 문구 확인
- [ ] 월 유지보수 범위 문서화

## 바로 팔기 좋은 패키지
1. 문의 자동화 세팅: 30만 원부터
2. 상담 인박스 세팅: 90만 원부터
3. CRM 파이프라인 세팅: 90만 원부터
4. 분석 대시보드 세팅: 90만 원부터
5. 내부 운영툴 세팅: 120만 원부터
