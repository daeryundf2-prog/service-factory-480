# 비영리 분석 대시보드 서비스

서비스 번호: 072
원본 ID: svc-448
rank / tier / score: 72 / B / 87
카테고리 / 니치: analytics / 비영리
권장 채널: web-service
플랫폼: Stripe Payment Links
결제 시스템: Stripe Payment Links
가격 앵커: ₩600,000 setup + ₩180,000/mo

## 오퍼
Metabase 또는 로컬 도구를 활용해 비영리의 분석 대시보드 흐름을 세팅하고 운영 문서까지 납품한다.

## 고객 문제
콘텐츠 업로드 소재가 꾸준히 나오지 않는다

## 첫 실행 기준
비영리의 현재 업무 화면 1개와 샘플 데이터 10개를 받아 첫 MVP 범위를 고정한다.

## 납품물
- 전환 이벤트 사전
- 데이터 품질 점검표
- 월간 리포트 템플릿
- KPI 정의 워크시트
- 대시보드 화면 구성표
- 운영 인수인계 체크리스트

## 작업 단계
- 1. 상담 전환 이벤트와 매출/문의 경로를 정의한다.
- 2. 데이터 원천, 필드명, 갱신 주기, 누락 기준을 점검한다.
- 3. 지표별 대시보드와 월간 리포트 템플릿을 세팅한다.
- 4. 고객 담당자에게 갱신, 해석, 수정 절차를 인수인계한다.

## 필수 보강
전환 이벤트 사전, 데이터 품질 점검표, 월간 리포트 템플릿을 추가한다.

## 약점과 방어
데이터 원천 품질과 전환 이벤트 정의가 없으면 보고서가 숫자 나열로 끝난다.

## 결제/채널 주의
- Stripe Payment Links: 서비스 착수금, 월 유지보수, 단일 결제 링크. 주의: 디지털 앱 기능을 모바일 앱 안에서 팔 때는 앱스토어 정책을 따로 본다.
- Stripe Checkout: 자체 웹서비스의 장바구니/구독. 주의: 개발 연동과 웹훅 검증이 필요하다.
- Paddle Checkout: 글로벌 디지털 상품/소프트웨어. 주의: 판매자 승인과 상품 심사가 필요하다.
- Lemon Squeezy Checkout: 라이선스/다운로드/구독. 주의: 계정 승인과 지원 국가 확인 필요.
- web: 정적 카탈로그 → 결제 링크 → 납품 ZIP/온보딩 폼 → 월 유지보수. 주의: 실제 결제 링크는 사용자 소유 계정에서 생성한다.

## 환불 및 성과 안내
실제 결제는 사용자 소유 계정에서만 진행합니다. 납품 범위, 환불 조건, 고객 자료 제공 지연 시 일정 변경 조건을 사전에 안내합니다. 이 서비스는 세팅과 운영 문서를 납품하지만 매출, 문의, 수익을 보장하지 않습니다.
